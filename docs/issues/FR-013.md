# [FR-013] X-Cross and Advanced Start-State Trainer

## Labels
`Future`, `product`, `cross`, `advanced-training`

## User Story
As a cuber, I want guided X-cross and other advanced start-state drills, so that I can practice more advanced opening efficiency intentionally.

## Proposed Solution

### Overview
Expand the cross training track into advanced opening trainers such as X-cross and niche start-state drills once the basic cross workflow is solid.

### Implementation Flow
1. Define advanced opening drill categories and success signals.
2. Determine what should be sourced from solver data versus curated examples.
3. Integrate advanced drills into the same training-plan and review model.

### Technical Approach
Build on the basic cross-drill model, but delay advanced optimization logic until the baseline trainer is proven.

### Key Considerations
- X-cross is a meaningful escalation in difficulty and data needs.
- This feature should not force the v1 cross trainer to overfit around advanced users.

## Acceptance Criteria

- [ ] X-cross is captured explicitly as future roadmap work.
- [ ] Dependencies on taxonomy, curated cases, and possibly solver support are noted.
- [ ] V1 cross scope remains intentionally simpler.
