# [FR-004] Cross Drill Workflows

## Labels
`MUS`, `product`, `cross`, `drills`

## User Story
As a cuber, I want dedicated cross drills with focused goals, so that I can improve planning and execution outside of full solves.

## Proposed Solution

### Overview
Add v1 planning for cross-specific drill blocks instead of limiting the trainer to last-layer cases.

### Implementation Flow
1. Define supported cross drill types for v1.
2. Define what completion and review data the trainer should capture.
3. Connect cross drills into shared plan and session structures.

### Technical Approach
Keep cross training within the same training-plan model as last-layer work, but avoid overpromising deep automatic grading in v1.

### Key Considerations
- Cross work should complement, not replace, full-solve practice.
- V1 should avoid pretending it can measure more than the current app can observe.

## Acceptance Criteria

- [x] Supported v1 cross drill types are explicitly named and bounded, including what the user is asked to do in each drill. ✅ Implemented with 5 cross drill types (CROSS-WHITE, CROSS-YELLOW, CROSS-CN, CROSS-PLAN, CROSS-EXEC) in case-catalog-data.js and crossDrillTypes map in trainer-active-session.js
- [x] Cross drills define which data can actually be captured in v1 (for example attempt count, self-rated success, completion time, or skipped drill) and avoid unsupported automatic grading claims. ✅ Cross drills capture attempt count, completion time, skip count, and DNF rate through the same recordAttempt flow as last-layer drills. No automatic grading claims — executionRequired flag distinguishes plan-only vs execution drills.
- [x] Shared plan/session structures identify the specific fields or block types that allow cross drills to coexist with last-layer drills. ✅ Cross drills use the same TrainingPlan, DrillQueue, and session result structures. Goal "cross" triggers cross-specific block sizing in planner-core.js with full-solve prompt mode and planTime targeting.
- [x] Review output for cross practice specifies the exact recap fields shown after a session and how they map to captured cross-drill data. ✅ Session review surface (`session-review.js`) displays cross drill results with case names, completion times, vs-avg comparisons, status tags, and next-step recommendations derived from recorded results. Review handoff preserves the `cross` goal so weak-case follow-up routes back into the cross setup flow instead of last-layer setup.
