# [FR-011] Multi-Puzzle Step Training Library

## Labels
`Future`, `product`, `curriculum`, `multi-puzzle`

## User Story
As a cuber, I want structured training sessions for many steps across multiple puzzles, so that the trainer can grow beyond 3x3 last layer and cross over time.

## Proposed Solution

### Overview
Extend the trainer from a focused v1 into a broader curriculum system that can support additional steps and puzzles without rewriting the trainer model each time.

### Implementation Flow
1. Define how puzzle, method, and step should be represented in the domain model.
2. Define rollout stages for adding new puzzle families and step libraries.
3. Reuse the same plan, drill, and review concepts across broader content.

### Technical Approach
Keep the current trainer contracts generic enough that future puzzle/step additions are catalog and UI extensions, not structural rewrites.

### Key Considerations
- This should not bloat v1.
- Additional puzzles should be added in curated waves, not all at once.

## Acceptance Criteria

- [ ] Future multi-puzzle growth is explicitly represented in the roadmap.
- [ ] Domain concepts are compatible with broader puzzle and step support.
- [ ] The feature remains future-scoped and does not expand v1.
