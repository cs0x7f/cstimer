# [FR-003] Adaptive Last-Layer Drill Queue

## Labels
`MUS`, `product`, `logic`, `last-layer`

## User Story
As a cuber, I want PLL and OLL drills to be weighted toward weak cases, so that my practice time targets the cases costing me the most.

## Proposed Solution

### Overview
Plan a drill queue that can bias repetitions toward weak cases while still covering the broader case set.

### Implementation Flow
1. Define the v1 last-layer case catalog.
2. Define weakness inputs and weighting rules.
3. Define queue-generation behavior for active sessions.

### Technical Approach
Represent weakness separately from raw solve storage and compute queue priority from trainer stats plus catalog metadata.

### Key Considerations
- Weak-case bias should not starve all other cases.
- Logic must remain explainable for session review output.

## Acceptance Criteria

- [ ] PLL (21 cases) and OLL (57 cases) training sets are explicitly enumerated with case IDs.
- [ ] Weighting logic documents which metrics drive case priority (e.g., solve time, recognition time, DNF rate, skip count).
- [ ] Queue behavior specifies minimum coverage guarantees (e.g., no case excluded for more than N consecutive selections).
- [ ] The weighting formula is documented clearly enough that an implementation agent can reproduce it without guessing.
