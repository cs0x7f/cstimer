# [FR-014] Virtual Cube Practice Surface

## Labels
`Future`, `product`, `virtual-cube`, `ui`

## User Story
As a cuber, I want a virtual cube that follows the scramble and case setup, so that I can practice niche cases even when I do not have a cube in hand.

## Proposed Solution

### Overview
Add a virtual-cube-assisted practice surface that makes case rehearsal and niche setup work easier than doing moves mentally from a static scramble drawing.

### Implementation Flow
1. Define the intended practice flows for a virtual cube.
2. Determine which existing csTimer visual or twisty capabilities can be reused.
3. Define where virtual-cube practice belongs in the trainer UI and roadmap.

### Technical Approach
Treat this as a future interaction layer that may reuse existing twisty/rendering pieces, but do not make it a prerequisite for the first trainer release.

### Key Considerations
- This is highly UI-bearing and needs mockup-first planning.
- It can become a powerful practice tool, but it should not delay the core trainer loop.

## Acceptance Criteria

- [ ] Virtual-cube practice is explicitly represented in the roadmap.
- [ ] Its dependency on UI design and interaction planning is called out.
- [ ] The feature remains out of v1 scope.
