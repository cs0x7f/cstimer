# [FR-002] Structured Training Plans

## Labels
`MUS`, `product`, `trainer-plans`

## User Story
As a cuber, I want reusable training templates and guided setup, so that I can practice with a plan instead of inventing sessions manually.

## Proposed Solution

### Overview
Define guided v1 templates that turn broad goals into concrete session structures.

### Implementation Flow
1. Specify supported plan archetypes for v1.
2. Define setup inputs such as session length and training focus.
3. Describe the output plan structure used by the drill queue.

### Technical Approach
Use domain-level `TrainingPlan` and `DrillDefinition` concepts that can later migrate out of csTimer.

### Key Considerations
- Keep plan setup simple enough for returning cubers.
- Avoid exposing too many knobs in v1.

## Acceptance Criteria

- [ ] V1 templates are defined and tied to the supported scope.
- [ ] Setup inputs are documented and bounded.
- [ ] Plan output shape is clear enough for a later implementation agent.
