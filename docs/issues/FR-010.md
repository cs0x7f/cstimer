# [FR-010] Multiplayer and Live Coaching

## Labels
`Future`, `product`, `multiplayer`

## User Story
As a cuber, I want shared live sessions, multiplayer coaching flows, and possibly integrated call-based collaboration, so that training can become collaborative later.

## Proposed Solution

### Overview
Reserve multiplayer and live social training as later product tracks once the single-player training layer is proven.

### Implementation Flow
1. Define what multiplayer means product-wise before choosing infrastructure.
2. Reuse future sync/profile concepts as prerequisites.
3. Decide whether live collaboration is lightweight presence, async challenge play, or voice/video room-based sessions.
4. Keep v1 focused on high-quality solo training flows.

### Technical Approach
Do not allow future multiplayer or integrated calling features to distort the current trainer domain and UI decisions for v1.

### Key Considerations
- Multiplayer is exciting but dangerous for scope.
- Built-in voice/video is even more infrastructure-heavy and must remain a later-layer decision.
- Solo training clarity must come first.

## Acceptance Criteria

- [ ] Multiplayer is explicitly future-only.
- [ ] The v1 plan is not blocked by multiplayer assumptions.
- [ ] Future dependencies are noted without inflating current implementation work.
