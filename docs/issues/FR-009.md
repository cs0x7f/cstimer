# [FR-009] Cloud Sync and Profiles

## Labels
`Future`, `platform`, `sync`

## User Story
As a cuber, I want optional profile sync across devices, so that I can keep trainer progress beyond one browser.

## Proposed Solution

### Overview
Plan future sync as an optional online layer on top of a local-first trainer.

### Implementation Flow
1. Define future profile and sync concepts.
2. Define the role of queued sync events.
3. Keep offline-first local usage as the primary source of truth.

### Technical Approach
Treat Convex or another backend as a future synchronization layer instead of a prerequisite for the product.

### Key Considerations
- Offline-first guarantees must survive sync adoption.
- Sync conflict handling should be designed before implementation later.

## Acceptance Criteria

- [ ] Sync is clearly marked future-only.
- [ ] Local-first behavior remains the base model.
- [ ] Future sync concerns are separated from v1 implementation work.
