# [FR-001] Trainer Entry and Goal Selection

## Labels
`MUS`, `product`, `ui`, `trainer-entry`

## User Story
As a cuber, I want a clear trainer entry point and goal-selection flow, so that I can start guided practice without leaving csTimer.

## Proposed Solution

### Overview
Add a trainer-facing entry surface that frames the feature as structured practice instead of another hidden tool toggle.

### Implementation Flow
1. Define where trainer mode is discovered in the current csTimer UI.
2. Present goal-first options such as returning cuber, last-layer focus, and cross focus.
3. Route the selection into training-plan setup without affecting classic timer flows.

### Technical Approach
Use an additive UI entry and avoid changing default timer behavior for users who never enter trainer mode.

### Key Considerations
- Discovery should feel native to csTimer.
- Classic solve flow must remain untouched unless trainer mode is explicitly entered.

## Acceptance Criteria

- [x] A trainer entry surface is defined in planning and design artifacts. ✅ Implemented in `src/js/trainer/trainer-entry-home.js`
- [x] Goal selection options map to the v1 training scope. ✅ Three goal cards: Return to Speed, Last Layer, Cross
- [x] Entry into trainer mode is explicit and does not hijack normal timer usage. ✅ Mounted through shared trainer shell, separate from timer flow
