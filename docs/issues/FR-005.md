# [FR-005] Skill Stats and Session Review

## Labels
`MUS`, `product`, `stats`, `review`

## User Story
As a cuber, I want post-session summaries and weakness tracking, so that I can see what improved and what still needs work.

## Proposed Solution

### Overview
Plan review surfaces that turn training data into understandable outcomes and next recommendations.

### Implementation Flow
1. Define `SkillStats` and `TrainingSessionResult` outputs.
2. Define which signals appear in session review.
3. Define persistent weakness-summary surfaces beyond a single session.

### Technical Approach
Build on existing case-stat ideas where useful, but keep trainer review output as a first-class planning concern.

### Key Considerations
- Review should be understandable, not just data-dense.
- Weakness ranking and session recap must align with the weighting model.

## Acceptance Criteria

- [x] Session review specifies which metrics appear (e.g., avg time, best/worst cases, improvement trend, weakness ranking). ✅ Implemented with stats overview, confidence panels, case rankings table, and trend sparklines
- [x] Persistent weakness summary defines what data persists across sessions and how it is computed. ✅ Computed from `SkillStats` stored in trainer persistence, with classification into Mastered/Learning/Weak/Untrained
- [x] Review outputs map to specific data fields produced by FR-003 weighting logic and FR-004 cross drill tracking. ✅ Uses `getAllStats()`, `loadSessions()`, and `getCatalog()` from trainerIntegration
- [x] At least one "next recommended action" recommendation is defined based on review data. ✅ "Drill Weak Cases" button and per-case "Drill" actions in ranking table
