# [FR-005] Skill Stats and Session Review

## Labels
`MUS`, `product`, `stats`, `review`

## User Story
As a cuber, I want post-session summaries and weakness tracking, so that I can see what improved and what still needs work.

## Proposed Solution

### Overview
Plan review surfaces that turn training data into understandable outcomes and next recommendations.

### Implementation Flow
1. Define `SkillStats` and `TrainingSessionResult` outputs.
2. Define which signals appear in session review.
3. Define persistent weakness-summary surfaces beyond a single session.

### Technical Approach
Build on existing case-stat ideas where useful, but keep trainer review output as a first-class planning concern.

### Key Considerations
- Review should be understandable, not just data-dense.
- Weakness ranking and session recap must align with the weighting model.

## Acceptance Criteria

- [ ] Session review content is specified.
- [ ] Persistent weakness summary content is specified.
- [ ] Review outputs line up with training logic and supported data.
